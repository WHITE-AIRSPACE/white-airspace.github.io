在移动端开发的时候，如果输入内容 不复杂的情况下，自己实现一个键盘，
这样可以规避很多问题，比如安全问题，input输入框调起键盘布局被影响的问题。

最终实现效果：

首先键盘布局

<ul class="keybord">
        <li
            v-for="(value, index) in letter"
            :key="index"
            :style="{ width: value === 0 ? '66%' : '32%' }"
        >
            {{ value }}
        </li>
    </ul>

	data() {
		return {
			letter: [1, 2, 3, 4, 5, 6, 7, 8, 9, 0, 'x'],
		};
	},


.keybord {
	width: 100%;
	margin-top: 30px;
	display: flex;
	justify-content: space-between; 
	flex-direction: row;
	flex-wrap: wrap;
}
.keybord li {
	width: 32%;
	height: 96px;
	margin-bottom: 10px;
	background: white;
	text-align: center;
	font-family: Helvetica;
	font-size: 50px;
	border-radius: 8px;
	color: #333333;
	line-height: 96px;
}

以上就实现了一个简单的键盘，加上点击事件就能使用了。另外删除按钮如果不使用 “X” 的话可以单独找个图处理。
